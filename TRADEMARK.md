# Trademark Policy

"Software Compass" and the associated logo (collectively, the "Marks") are
identifiers of this project and its official releases. This policy is
separate from, and in addition to, the code license in [`LICENSE`](./LICENSE)
(AGPL-3.0-only). Open-sourcing the code does not include a license to use the
project's name or branding.

## What you may do without asking

* Use the Marks to refer to this project truthfully, e.g. "built with
  Software Compass", "compatible with Software Compass", in blog posts,
  talks, documentation, or comparisons.
* Use an unmodified copy of the software internally, and refer to it by its
  original name, as permitted by the [`LICENSE`](./LICENSE).
* Fork the project (subject to the license terms), as long as your fork is
  clearly distinguished from the original (see below).

## What requires a different name

You may **not** use "Software Compass" or a confusingly similar name/logo for:

* A modified, forked, or rebranded version of the software offered in a way
  that could be mistaken for an official release.
* Any product, service, hosted offering, plugin marketplace, or company name
  that implies official affiliation with, sponsorship by, or endorsement from
  the Software Compass maintainers, when none exists.
* Domain names, social media handles, or app store listings that could
  confuse users into thinking they are interacting with the official project.

If you fork or redistribute a modified version, choose a clearly different
name (e.g. "MyToolName, based on Software Compass") and state
that it is an unofficial, unaffiliated derivative.

## Rationale

This policy exists to prevent user confusion and to stop third parties from
trading on the reputation of this project — for example, offering a
rebranded or lightly modified copy as if it were the official product or an
official commercial offering. It does not restrict truthful use, forking, or
contributing back under the project's license.

## Questions

If you are unsure whether your intended use is covered, please open a
GitHub issue or contact the maintainers before using the Marks.
